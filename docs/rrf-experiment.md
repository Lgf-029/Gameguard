# RRF 融合对比实验记录

## 一、实验目的

验证三路检测（规则引擎 + 向量检测 + 图检测）在 RRF 融合后的召回率提升效果，量化每一路检测对最终风控决策的贡献。

## 二、实验环境

- 数据：init_data.py 生成的 300 条交易记录（290 正常 + 10 异常）
- 标注：10 条异常交易的 transaction_id 和 risk_label 已知（fraud）
- 评估指标：Recall@5（风控场景漏报比误报更致命）

## 三、实验方案

对同一批 300 条交易数据，依次运行三组对照实验：

1. 仅规则引擎
2. 规则引擎 + 向量检测
3. 规则引擎 + 向量检测 + 图检测（三路 RRF 融合）

每组实验统计 Top-5 高风险交易中命中的真实异常交易数量，计算 Recall@5。

## 五、结果分析

仅规则引擎能抓到大部分显式异常（如新注册大额充值、设备农场），但对隐蔽性强的变种行为（如模拟器伪装、低频渐进式欺诈）存在漏报。加入向量检测后，Milvus 根据历史异常行为的语义相似度补充了一部分规则引擎未能覆盖的案例，召回率从       提升至      。再加入图检测后，Neo4j 通过设备-账号关系网络发现了共享设备和设备农场异常，进一步将召回率提升至       。

## 六、局限性说明

当前实验基于 300 条模拟数据，绝对值不代表生产环境效果。生产环境中可通过以下方式迭代：

- 积累真实异常案例，扩大标注数据集
- 根据 Bad Case 分析调整各检测器的权重和阈值
- 引入生产环境真实噪声（正常交易中的边界情况）做压力测试

## 七、结论

三路并行检测 + RRF 融合在模拟数据上验证了多模态风控的有效性。各检测器互相补充，每一路的加入都带来了可量化的召回率提升。生产环境中将持续通过 Bad Case 分析和标注数据迭代来优化检索策略。
